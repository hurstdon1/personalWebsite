Jam Session

Jam Session is a full-stack application representing a minimum viable product (MVP) of a LinkedIn type social media hub for musicians to find each other, and find people to JAM with and for finding musicians and artists via social media-like functionality.

The Jam Session app is:

  • A responsive full-stack web application that allows users to create an account, set up a profile, create groups, post within those groups, search for musicians based on preferred criteria, post gigs, and send direct messages to other users
  • Defined, designed, and created many custom features whilst utilizing a bevy of Django packages
  • Designed and configured database and back-end applications
  • Built using a variety of technologies including Python/Django, PostgreSQL, AWS, Heroku, HTML, CSS, and Javascript

Keyword Generator

The Keyword Generator Program was something I created as part of a group project for a software engineering course. The project uses a series of REST APIs to start with the name of a video game before ultimately delivering a word/image cloud of words associated with the game.

My portion of the project does the following:

  • Takes user input in the form of a text document
  • This input can be sent via an api route or uploaded directly to the site via a dropzone
  • The program then breaks apart the json data and extracts a list of keywords based on their weight
  • Built using Python/Flask, and SQL

Small Shell

The Small Shell program is an implementation of a BASH-like shell

It has the following functionality

  • The shell command program takes user input and returns results similar to other popular shells
  • Commands can include blank space, multiple flags, and input/output redirection
  • Commands can be executed in the foreground & background
  • Supports signal interrupts which can be sent from the keyboard to interrupt processes

XiangQi Game

The Xiangqi Game program is an implementation of an abstract board game.

It has the following functionality

  • Initializes a game board and fills it with unique pieces
  • Allows the user to make moves by calling a function and passing in the starting/ending locations
  • The program checks that the move is valid and if it is, implements the move
  • The program checks if each move will put the user in check or has put the opponent in checkmate
